## 📝 테크니컬 라이팅 에세이 (Essays)

#### Q. 본인이 생각하는 테크니컬 라이터의 역할은?
> 복잡한 기술과 정보를 구조화하여 제품의 신뢰도를 높이고, 팀원들과 외부 파트너가 오해 없이 빠르게 소통할 수 있도록 '싱글 소스(SSOT)'를 구축해 팀의 전체적인 일하는 속도를 올리는 사람이라고 생각합니다.

#### Q. 문서화 자체에 재미를 느꼈던 순간은?
> 매뉴얼 제작 중 팀원들이 반복되는 XML/DITA 변환 오류로 인해 일정이 지연되는 문제를 발견했을 때, 스스로 에러 패턴을 분석하고 대응 방법을 리스트로 정리하여 공유했던 순간입니다. 내가 만든 문서(가이드라인) 덕분에 빡빡한 스케줄 속에서도 팀원들이 헤매지 않고 품질 높은 산출물을 기한 내에 성공적으로 뽑아내는 것을 보며, 잘 만든 문서 하나가 조직의 생산성을 극대화할 수 있다는 것에 큰 보람과 재미를 느꼈습니다.
